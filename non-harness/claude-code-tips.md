# Claude Code 사용 팁 모음

Claude Code(터미널/IDE/데스크톱에서 동작하는 AI 코딩 에이전트)를 효율적으로 쓰기 위한 실전 팁을 정리한 문서입니다.

---

## 1. 기본 입력 & 단축키

| 동작 | 방법 |
|------|------|
| 메시지 전송 | `Enter` |
| 줄바꿈(여러 줄 입력) | `\` + `Enter` 또는 `Option+Enter`(Mac) |
| 직전 메시지 수정 | `↑` 화살표로 히스토리 탐색 |
| 진행 중 작업 중단 | `Esc` (한 번) |
| 직전 턴으로 되돌리기 | `Esc` 두 번 → 이전 메시지 선택 |
| 자동완성 | `Tab` (파일 경로 등) |
| 셸 명령 직접 실행 | 프롬프트에 `!` 접두사 — 예: `!git status` |
| 슬래시 명령 | `/` 입력 후 명령 선택 |

> **팁**: 로그인처럼 사람이 직접 해야 하는 인터랙티브 명령(`gcloud auth login` 등)은 `! <명령>`으로 실행하면 출력이 대화에 바로 들어와 Claude가 이어서 처리할 수 있습니다.

---

## 2. 컨텍스트 관리 (가장 중요)

Claude Code의 성능은 **컨텍스트 위생(context hygiene)** 에 크게 좌우됩니다.

- **`/clear`**: 작업이 끝나거나 주제가 완전히 바뀌면 컨텍스트를 비웁니다. 긴 무관한 히스토리는 응답 품질을 떨어뜨립니다.
- **`/compact`**: 대화가 길어졌지만 맥락을 유지하고 싶을 때 요약 압축합니다. (자동으로도 일어남)
- **한 작업 = 한 세션** 원칙: 버그 수정과 새 기능 개발을 한 세션에 섞지 마세요.
- 큰 파일 전체를 무턱대고 읽히기보다 **필요한 부분/심볼만** 지정해 주세요.

---

## 3. CLAUDE.md — 프로젝트 규약 파일

프로젝트 루트(또는 `~/.claude/CLAUDE.md` 전역)에 두면 매 세션 자동 로딩됩니다.

```markdown
## 코딩 규약
- 들여쓰기: 스페이스 2칸
- 커밋 메시지: 한국어로 작성
- 테스트: 변경 시 항상 `npm test` 실행

## 프로젝트 구조
- src/ : 소스
- tests/ : 테스트
```

- `/init` 명령으로 코드베이스를 분석해 초안을 자동 생성할 수 있습니다.
- **자주 반복 설명하는 내용**(빌드 명령, 컨벤션, 금지사항)은 여기에 적어두면 매번 안 알려줘도 됩니다.
- 우선순위: **사용자 지시(CLAUDE.md) > 스킬 > 기본 동작**.

---

## 4. 계획 모드 (Plan Mode)

복잡한 작업은 **먼저 계획부터** 세우게 하면 실수가 줄어듭니다.

- 큰 기능/리팩토링은 "바로 코딩하지 말고 계획부터 보여줘"라고 요청하세요.
- 계획을 검토·승인한 뒤 실행에 들어가면, 방향이 어긋나는 것을 미리 막을 수 있습니다.
- `Shift+Tab`으로 권한 모드(plan / 자동 수락 등)를 전환할 수 있습니다.

---

## 5. 권한 모드 활용

- **기본 모드**: 파일 쓰기·명령 실행 시마다 확인을 요청합니다(안전).
- **자동 수락(acceptEdits)**: 신뢰할 수 있는 반복 작업에서 편집을 자동 승인합니다.
- 자주 쓰는 안전한 명령(`git status`, `ls` 등)은 한 번 허용 후 **항상 허용**으로 설정하면 프롬프트가 줄어듭니다.
- `/fewer-permission-prompts` 류의 도구로 자주 쓰는 read-only 명령을 일괄 허용 목록에 추가할 수 있습니다.

---

## 6. 서브에이전트 & 병렬 작업

- **독립적인 작업 여러 개**는 병렬로 던지면 빠릅니다 (예: "A 모듈 분석과 B 모듈 분석 동시에").
- 넓은 범위 탐색(여러 파일·디렉터리 훑기)은 **탐색 전용 서브에이전트**에게 위임하면 메인 대화 컨텍스트가 깨끗하게 유지됩니다 — 결론만 받아오기 때문.
- 큰 구현·테스트 작성·여러 파일 리팩토링은 서브에이전트 위임에 적합합니다.

---

## 7. 자주 쓰는 슬래시 명령

| 명령 | 용도 |
|------|------|
| `/clear` | 컨텍스트 초기화 |
| `/compact` | 대화 압축 |
| `/init` | CLAUDE.md 자동 생성 |
| `/review` | PR/변경사항 리뷰 |
| `/config` | 설정(테마, 모델 등) 변경 |
| `/help` | 도움말 |

---

## 8. 효과적인 프롬프트 작성법

1. **구체적으로**: "버그 고쳐줘" ❌ → "로그인 시 토큰이 갱신 안 되는 버그를 `auth.ts`에서 찾아 고쳐줘" ✅
2. **기대 결과 명시**: "테스트 통과까지 확인해줘", "변경 파일 목록과 diff 요약을 알려줘"
3. **제약 조건 전달**: "외부 라이브러리 추가 없이", "기존 패턴을 따라서"
4. **단계 분리**: 큰 작업은 "먼저 X만, 검토 후 Y" 처럼 나눠서.
5. **검증 요구**: "정말 동작하는지 실행해서 확인해줘" — 추측이 아닌 증거 기반 응답을 유도.

---

## 9. 이미지 & 파일 참조

- 스크린샷을 붙여넣으면(클립보드 → `Ctrl+V`) UI 버그, 에러 화면을 그대로 보여줄 수 있습니다.
- 코드 위치는 `파일경로:줄번호` 형식이 클릭 가능합니다 — 예: `src/app.ts:42`.
- `@파일경로`로 특정 파일을 명시적으로 참조할 수 있습니다.

---

## 10. 안전 수칙

- **되돌리기 어려운 작업**(파일 삭제, 강제 push, 외부 전송)은 항상 먼저 확인을 요청하게 두세요.
- Claude가 "완료했다"고 하면, **검증 명령 출력**을 함께 요구하세요. 테스트 실패는 실패라고 말하도록.
- 커밋·푸시는 명시적으로 요청할 때만 하도록 합니다.
- 민감 정보(키, 비밀번호)는 프롬프트에 직접 넣지 말고 환경변수·시크릿 매니저를 쓰세요.

---

## 11. MCP & 확장

- **MCP(Model Context Protocol)** 서버로 외부 도구(브라우저 자동화, DB, 문서 검색 등)를 연결할 수 있습니다.
- **스킬(Skill)**: 특정 작업에 대한 전문 절차/지식. `/<스킬명>`으로 호출합니다.
- **context7** 같은 MCP로 최신 라이브러리 공식 문서를 실시간 조회 — 학습 데이터가 오래됐을 수 있는 라이브러리 API에 유용.

---

## 12. 워크플로우 추천 패턴

```
1. /clear 로 시작            (깨끗한 컨텍스트)
2. 작업 설명 + 계획 요청      (Plan Mode)
3. 계획 검토·승인
4. 실행 (필요시 자동 수락)
5. 검증 요청 (테스트·실행)
6. 커밋 요청 (한국어 메시지)
7. /clear 후 다음 작업
```
