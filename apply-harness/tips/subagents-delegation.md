# 무거운 작업을 서브에이전트에 위임하기

## 상황
"이 키워드가 어디서 쓰이는지 다 찾아줘", "여러 모듈을 읽고 요약해줘"처럼 많은 파일을 훑어야 하는 작업을 메인 대화에서 직접 하면 파일 내용이 컨텍스트를 가득 채운다. 결론만 필요할 때는 서브에이전트에 맡긴다.

## 방법
1. 넓은 탐색·다중 파일 작업은 서브에이전트에 위임한다. 메인은 결론만 받는다.
2. 서로 의존 없는 작업이 2개 이상이면 **병렬로 동시에** 디스패치한다(한 번에 여러 에이전트 호출).
3. 서브에이전트는 메인 대화를 모르므로, 필요한 맥락·경로·규약을 위임 프롬프트에 명시한다.

## 예시
병렬 위임 시나리오:
```text
- 에이전트 A: 인증 모듈에서 토큰 검증 로직 위치 찾기
- 에이전트 B: 결제 모듈의 에러 처리 패턴 요약
→ 두 작업은 독립적이므로 한 번에 동시에 디스패치하고,
  각자 결론(파일 경로/요약)만 회수한다.
```

위임 프롬프트에 포함할 것:
- 원 요청 + 목표
- 작업 디렉터리/대상 파일 범위
- "프로젝트 CLAUDE.md 규약을 준수하라" 같은 제약
- 받고 싶은 산출물 형식(변경 파일 목록, diff 요약 등)

## 주의점
- 한 줄 수정, 단순 질문 같은 작은 일까지 위임하면 오히려 느려진다. 무겁고 탐색적인 일에만 쓴다. → [[subagents-model-routing]]
- 같은 파일을 여러 에이전트가 동시에 수정하면 충돌한다. 쓰기 작업은 범위를 겹치지 않게 나눈다.
- 백그라운드 에이전트는 권한 프롬프트에 응답할 수 없어, 사전 허용되지 않은 도구(예: Write)에서 막힐 수 있다. 필요한 권한을 미리 settings에 허용해 둔다. → [[settings-permissions]]
