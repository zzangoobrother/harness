# 에이전트별 모델 지정으로 비용·속도 분기

## 상황
모든 작업을 최고 성능 모델로 돌리면 비용·시간이 낭비된다. 작업 성격에 따라 모델을 나누면 효율이 좋아진다.

## 방법
에이전트 정의의 프론트매터 `model` 필드, 또는 위임 시점에 모델을 지정한다. 대략적인 분기 기준:

- **무거운 구현·테스트 작성·리팩토링·여러 파일 생성** → `sonnet` 같은 중급 모델에 위임
- **설계 논의·코드 리뷰·디버깅 상담·작은 편집** → 메인의 고성능 모델(`opus` 등)에서 직접 처리
- **단순·대량 반복 작업** → `haiku` 같은 빠른 모델

## 예시
```markdown
---
name: refactorer
description: 여러 파일에 걸친 리팩토링 실행 에이전트.
tools: Read, Edit, Grep, Glob, Bash
model: sonnet
---
```

위임 호출 시 모델만 바꿔 같은 작업을 더 싸게/빠르게 돌릴 수도 있다(메인은 opus 유지, 실행은 sonnet 위임).

## 주의점
- 모델을 낮추면 복잡한 추론 품질이 떨어질 수 있다. 설계·리뷰처럼 판단이 중요한 일은 고성능 모델에 둔다.
- 위임 기준은 팀 규약(CLAUDE.md)에 명문화해 두면 일관성이 생긴다. → [[claude-md-memory]]
- 확인 필요: 지정 가능한 정확한 모델 별칭(sonnet/opus/haiku 등)과 표기는 버전·플랜에 따라 다를 수 있으니 확인한다.
