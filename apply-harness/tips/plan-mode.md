# 플랜 모드로 실행 전 계획 검토받기

## 상황
바로 코드를 고치기 전에 "무엇을 어떻게 바꿀지" 계획을 먼저 보고 승인하고 싶을 때 플랜 모드를 쓴다. 큰 변경이나 위험한 작업에서 사고를 줄인다.

## 방법
1. `Shift + Tab`으로 모드를 순환해 플랜 모드로 들어간다.
2. 요청을 입력하면 Claude가 파일을 **읽기만** 하며 변경 계획을 세워 제시한다(이 모드에서는 수정/실행을 하지 않는다).
3. 계획을 검토하고 승인하면 실제 실행으로 넘어간다. 수정이 필요하면 피드백을 줘 계획을 고친다.

## 예시
```text
Shift + Tab        # 플랜 모드 진입
> 결제 모듈에 재시도 로직을 추가해줘

# Claude가 변경 계획(대상 파일, 단계)을 제시
# → 검토 후 승인 → 실행
```

## 주의점
- 플랜 모드에서는 파일이 바뀌지 않으므로, 탐색·설계 단계에 안전하게 쓸 수 있다.
- 계획이 마음에 들지 않으면 승인하지 말고 피드백으로 방향을 잡는다. 한 번 승인하면 실행이 시작된다.
- 작은 수정에는 과한 절차다. 영향 범위가 넓거나 되돌리기 어려운 작업에 쓴다. → [[subagents-delegation]]
