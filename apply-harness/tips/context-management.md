# 컨텍스트 관리로 토큰 아끼기

## 상황
긴 세션에서 대화가 쌓이면 컨텍스트가 차고 토큰 비용이 올라가며 응답도 느려진다. 작업 전환 시점마다 컨텍스트를 정리하면 깔끔하게 이어갈 수 있다.

## 방법
- `/clear` — 컨텍스트를 완전히 비운다. **새로운 무관한 작업**으로 넘어갈 때 쓴다.
- `/compact` — 지금까지 대화를 요약 압축한다. 맥락은 유지하되 토큰을 줄인다. **같은 작업을 이어갈 때** 쓴다.
- 무거운 탐색·다중 파일 읽기는 서브에이전트에 위임해 메인 컨텍스트 오염을 막는다. → [[subagents-delegation]]

## 예시
```text
# 기능 A 작업 끝 → 전혀 다른 기능 B 시작
/clear

# 같은 기능을 계속 작업 중인데 대화가 너무 길어짐
/compact
```

판단 기준:
```text
이전 맥락이 더 이상 필요 없다  → /clear
이전 맥락은 유지하되 가볍게    → /compact
```

## 주의점
- `/clear`는 되돌릴 수 없다. 진행 중 작업 맥락이 사라지므로 확신이 없으면 `/compact`를 쓴다.
- 자동 압축이 일어날 때 `PreCompact` 훅으로 사전 처리를 끼울 수 있다. → [[hooks-overview]]
- 한 세션에서 여러 큰 작업을 섞지 말고, 작업 경계마다 `/clear`로 분리하면 품질이 안정적이다.
